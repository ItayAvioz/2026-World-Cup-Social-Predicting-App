06 — Scoring Engine
WorldCup 2026 · Architecture Explorer
Database React App
Steps
Layers
Database (PostgreSQL)
All scoring logic lives in DB triggers and RPCs. fn_update_points fires on game score update. fn_award_champion_points fires when final knockout_winner is set.
React App
Reads leaderboard via RPC. Displays points earned per game. No scoring logic runs in the frontend — it always reads from DB.
Supabase
Triggers fire on INSERT/UPDATE to games. RLS ensures users can read their own points. RPCs compute aggregate totals for leaderboard display.
